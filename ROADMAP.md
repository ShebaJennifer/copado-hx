# Future Vision & Roadmap — copado-hx

Copado-hx is designed not just as a CLI tool, but as a foundation for a developer-first DevOps platform for Copado.

The following roadmap outlines how this evolves into a fully autonomous and AI-driven DevOps system.


Highlight: MCP Server

Expose copado-hx as a Model Context Protocol (MCP) tool, allowing any AI agent to execute DevOps workflows autonomously.

---

## Near-Term (1-3 months)

| Phase | Feature | Description |
|-------|---------|-------------|
| **v0.2** | MCP Server | Expose copado-hx as a Model Context Protocol tool — any AI framework can call it natively |
| **v0.3** | Pipeline Visualization | ASCII flowchart showing the promotion path with real-time status at each stage |
| **v0.4** | Bulk Operations | Commit, promote, and deploy across multiple user stories in a single command |

## Mid-Term (3-6 months)

| Phase | Feature | Description |
|-------|---------|-------------|
| **v0.5** | CI/CD Plugin | GitHub Actions and Azure DevOps integration — run copado-hx in automated pipelines |
| **v0.6** | Multi-Org Support | Manage multiple Copado orgs from a single CLI session with `copado-hx org switch` |
| **v0.7** | Pipeline Templates | Codify common deployment patterns (hotfix, feature, release) as reusable workflows |

## Long-Term Vision

| Feature | Description |
|---------|-------------|
| **VS Code Extension** | Embedded terminal + sidebar with story context, inline test results, and AI chat |
| **Copado Native CLI** | Offer copado-hx as an official Copado CLI tool (like Salesforce's `sf` CLI) |
| **Webhook Listeners** | React to Copado platform events (job completion, test failures) in real-time |
| **Distributed Teams** | Shared CLI sessions with team-wide story context and deployment coordination |

This roadmap positions copado-hx as the bridge between Copado, CLI-based workflows, and AI-driven automation.
