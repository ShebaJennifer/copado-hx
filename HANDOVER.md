## FIRST — Getting Started

### Step 1: Open Terminal

Open PowerShell or Terminal in VS Code. Navigate to the project:
```powershell
cd "C:\Copado Hackathon\copado-hx"
```

### Step 2: Activate the Virtual Environment

```powershell
.venv\Scripts\activate
```

You should see `(.venv)` at the start of your terminal prompt. If you don't, the CLI won't work.

### Step 3: Check Everything Works

```powershell
copado-hx auth status
```

You should see something like:
```
  Salesforce OAuth Access Token:  Authenticated (****jhoF)
  Copado Actions API Key:         Authenticated (****6g==)
  CRT Personal Access Key:        Authenticated (****CB9M)
  Copado AI Platform API Key:     Not configured
```

**If you see "Not configured" for Salesforce OAuth**, the token has expired. Fix it:
```powershell
copado-hx auth login
```
This opens your browser. Log in to Salesforce, then **copy the full URL** from the browser address bar after login and **paste it back** into the terminal when asked.

### Step 4: Quick Smoke Test

Run the automated demo to make sure everything works end-to-end:
```powershell
copado-hx demo
```

This runs 8 steps automatically against the real Copado org. Takes about 20 seconds. If all steps show green checkmarks, you're good.

## THE 3 DEMO SCENARIOS

### SCENARIO 1: "Happy Path" — Commit to Deploy in 30 Seconds

**The Story:** A Salesforce developer completes their work and pushes it all the way to production — commit, promote, deploy, and auto-generated release notes — without ever opening a browser.

**Run these commands one by one:**

```powershell
# "Let me check what user stories I'm working on"
copado-hx story list
```
Wait for the table to appear. Then:

```powershell
# "Let me see the details of my story"
copado-hx story show --id US-0000024
```
Wait for the panel. Then:

```powershell
# "I've made my changes. Let me commit."
copado-hx commit --us US-0000024 -m "feat: lead scoring logic"
```
Wait for green checkmark. Then:

```powershell
# "Now let me promote to Integration"
copado-hx promote --us US-0000024 --env INT-SFP
```
Wait for green checkmark. Then:

```powershell
# "Let me deploy it"
copado-hx deploy --env INT-SFP --yes
```
Wait for green checkmark. Then:

```powershell
# "Now let me ask AI to write release notes for me"
copado-hx ai ask --agent release "Generate release notes for US-0000024 which adds lead scoring logic to prioritize inbound leads"
```
Wait ~10 seconds. AI will generate a full formatted release note with component tables, rollback plan, etc.

**Talking points for judges:**
- "6 commands. 30 seconds. No browser."
- "Every command hits a real Copado org — these are real Salesforce records being created."
- "The AI release notes are generated by Copado's Release Agent — real AI, not a template."

### SCENARIO 2: "Quality Gate" — Tests Block a Bad Deploy

**The Story:** Before deploying to production, we run automated tests. The tests reveal problems. Our Deployment Confidence Score tells the team it's NOT safe to deploy. Then AI analyzes the failures and tells us exactly what to fix.

**IMPORTANT:** This scenario uses mock mode because our trial org doesn't have CRT test cases with actual results. The mock data is realistic and demonstrates the feature perfectly.

**Step 1:** Turn on mock mode. Open the file `.copado-hx.json` in the project root and change:
```json
"mock_mode": true
```

**Step 2:** Run these commands:

```powershell
# "Let me check what test suites we have"
copado-hx test list
```
You'll see test suites including "Smoke Test Suite" (15 tests) and "Full Regression Suite" (87 tests).

```powershell
# "Let me run the smoke tests before deploying"
copado-hx test run --job job-smoke-001
```
You'll see "Test execution started" with an execution ID (e.g. `exec-abc12345`).

```powershell
# "Let me check the results" (use the execution ID AND job ID from above)
copado-hx test results --execution exec-abc12345 --job job-smoke-001
```

**IMPORTANT:** Both `--execution` and `--job` are required. The execution ID comes from the `test run` output. The job ID is the same one you used to run the test.

You'll see:
- **Result:** 14 passed, 1 failed (93% pass rate)
- **Deployment Confidence Score:** A visual bar showing the score
- **Recommendation:** Based on the score — "DEPLOY WITH CAUTION" or "DO NOT DEPLOY"
- **Failure detail:** Which test failed and why

```powershell
# "Let me ask AI to analyze why the test failed" (both --execution and --job required)
copado-hx ai triage --execution exec-abc12345 --job job-smoke-001
```
AI will show:
- Root cause analysis
- Recommended fix
- Risk assessment

**Step 3:** After demo, turn mock mode back off:
```json
"mock_mode": false
```

**Talking points for judges:**
- "The Deployment Confidence Score is our innovation feature — it combines test pass rate, failure severity, and coverage into a single go/no-go number."
- "When tests fail, AI automatically analyzes the failures and suggests fixes — no human debugging needed."
- "This is QA Intelligence — turning test data into deployment decisions."

### SCENARIO 3: "AI-Powered Development" — 5 Agents, One CLI

**The Story:** A developer uses Copado's AI agents to go from idea to deployment. Each agent specializes in a different phase of the DevOps lifecycle.

**Run these commands one by one:**

```powershell
# "I need to build an Apex trigger. Let me ask the Build Agent."
copado-hx ai ask --agent build "Write an Apex trigger for Contact that auto-populates Description on insert"
```
Wait ~10 seconds. AI generates: trigger + handler class + test class + deployment notes.

```powershell
# "Now let me ask the Test Agent to generate automated test scripts"
copado-hx ai ask --agent test "Generate a CRT QWord test script to verify Contact Description is set on insert"
```
Wait ~10 seconds. AI generates a CRT test script.

```powershell
# "Time to deploy. Let me get release notes from the Release Agent."
copado-hx ai ask --agent release "Generate release notes for the Contact Description auto-population feature"
```
Wait ~10 seconds. AI generates structured release notes.

```powershell
# "Finally, the Operate Agent creates my change management plan"
copado-hx ai ask --agent operate "Create a change management plan for rolling out the Contact Description trigger to production"
```
Wait ~10 seconds. AI generates training plan, documentation updates, monitoring plan.

**BONUS — Interactive Chat:**
```powershell
# "I can also have a conversation with any agent"
copado-hx ai chat --agent build
```
Type questions, get answers. Type `exit` to quit.

**Talking points for judges:**
- "5 specialist AI agents: Plan, Build, Test, Release, Operate — covering the entire DevOps lifecycle."
- "This is real Copado AI — streaming responses from the Copado GPT API, not mock data."
- "Any AI coding agent (Cursor, Claude, etc.) can drive these commands autonomously using our SKILL.md file."

## TROUBLESHOOTING

### "Token expired" or auth errors
```powershell
copado-hx auth login
```
Log in via browser, paste the URL back.

### Command not found: copado-hx
Make sure the venv is activated:
```powershell
.venv\Scripts\activate
```

### AI commands are slow
Normal — each AI response takes 5-15 seconds because it's streaming from the real Copado GPT API.

### Commands show mock data instead of real data
Check `.copado-hx.json` — make sure `"mock_mode": false` (except for Scenario 2 which uses mock mode).

### I messed something up and want to reset
Everything is in Git. Just run:
```powershell
git checkout .
```
